# Sprint Planner

Инструмент для планирования спринта: распределяй задачи по команде, раскладывай их на фазы и сразу видь, кто перегружен и где возникают конфликты.

---

## С чего начать

### 1. Добавь участников команды

Нажми кнопку **Участники** в правом верхнем углу. Введи имя и выбери роль — **Разработчик** или **Тестировщик**. Каждому участнику автоматически назначается цвет, по которому его строка будет видна на таймлайне.

Участников можно редактировать или удалять прямо из списка. При удалении участника он снимается с фаз всех задач.

### 2. Настрой спринт

Нажми на название спринта в шапке, чтобы изменить дату начала или длину. Спринт по умолчанию — 14 дней. Между спринтами можно переключаться стрелками `‹` `›` — задачи хранятся отдельно для каждого спринта.

### 3. Создай задачи

Нажми кнопку **+ Задача** или кликни прямо на ячейку таймлайна — задача сразу встанет на нужного человека и нужный день.

В форме задачи:

- **Название** — произвольное, например `PROJ-42 Авторизация`
- **Цель спринта** — отмечай задачи, критичные для спринта; они выделяются значком 🔥 и группируются отдельно в списке
- **Дата начала** — в пределах текущего спринта; выходные пропускаются автоматически
- **Фазы** — по умолчанию Dev / Review / Test; можно добавлять, удалять и менять порядок

### 4. Настрой фазы задачи

Каждая фаза — это отдельный исполнитель и оценка времени.

| Фаза | Поведение |
|------|-----------|
| **Dev** | Назначается на любого разработчика |
| **Test** | Доступны только тестировщики |
| **Review** | Исполнитель не указывается; фаза отображается на строке предыдущего участника и не создаёт конфликтов |

Оценку можно вводить в любом формате: `8h`, `2d`, `1d 4h` или просто `12` (воспринимается как часы). Рабочий день = 8 часов.

Фазы идут последовательно. Если нужно задать явный старт фазы — это делается через поле `startAfterDays` (пока только через код).

Внизу формы всегда показывается суммарная оценка и предполагаемая дата завершения. Если задача выходит за пределы спринта — форма подсветит это красным.

## Таймлайн

После сохранения задача появляется на таймлайне — горизонтальной сетке, где строки это люди, а столбцы — дни спринта.

- **Блок фазы** окрашен в цвет задачи; при наведении показывается название и длительность
- **Перетаскивание** — задачу можно двигать по таймлайну мышью; дата начала обновится автоматически
- **Клик на пустую ячейку** — открывает форму новой задачи, привязанной к этому человеку и дню
- **Клик на блок фазы** — открывает редактирование задачи

Выходные дни на сетке выделены другим фоном и пропускаются при расчёте длительности фаз.

---

## Конфликты и загрузка

Sprint Planner автоматически отслеживает перегрузку.

**Конфликт** возникает, когда суммарная нагрузка человека в один день превышает 8 часов (один рабочий день). Все фазы, участвующие в конфликте, подсвечиваются красным. В шапке появляется счётчик `⚠ N конфликтов`.

В левой панели задачи с конфликтом также помечаются значком `⚠`.

Review-фазы в подсчёт нагрузки не входят.

---

## Список задач (левая панель)

Слева всегда виден список задач текущего спринта. Задачи разбиты на две группы: **Цели спринта** и **Остальные задачи**.

- Поиск по названию — в реальном времени
- Клик на карточку — открывает редактирование
- Кнопка `×` при наведении — удаляет задачу

---

## Хранение данных

Все данные сохраняются в браузере (`localStorage`). Ничего не отправляется на сервер. При очистке данных браузера данные будут потеряны — при необходимости экспортируй состояние вручную через DevTools.

## Запуск локально

```bash
npm install
npm run dev
```

После запуска откройте адрес, который покажет Vite в терминале, обычно `http://localhost:5173`.

```bash
npm run build    # production-сборка → dist/
npm run preview  # предпросмотр сборки
```
